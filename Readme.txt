# Юни Мониторинг Серверов

Юни Мониторинг Серверов — это инструмент мониторинга системы, написанный на Python, который может отслеживать использование ЦП, памяти и диска и отправлять оповещения через Telegram.

## Использование

Сначала вам нужно создать бота Telegram и получить токен API. Вы можете обратиться к [официальной документации Telegram](https://core.telegram.org/bots/features#botfather), чтобы создать бота Telegram.

Затем вам нужно получить идентификатор чата Telegram, на который вы хотите получать оповещения. Вы можете использовать [@userinfobot](https://t.me/userinfobot), чтобы получить идентификатор чата.

Далее клонируйте этот проект:

```bash
git clone https://github.com/bboysoulcn/telemonitor.git
```

Вы можете использовать docker compose для запуска TeleMonitor:

```bash
docker-compose up -d
```

Или вы можете использовать k8s для запуска TeleMonitor:

```bash
kubectl apply -f deployment.yaml
```

Вот переменные среды:

- `TG_API_TOKEN`: API-токен вашего бота Telegram.
- `TG_CHAT_ID`: идентификатор чата Telegram, на который вы хотите получать оповещения.
- `TG_API_BASE_URL`: базовый URL-адрес API Telegram, по умолчанию `https://api.telegram.org`.
- `CPU_PERCENT`: пороговое значение загрузки ЦП. TeleMonitor отправит оповещение, если использование превысит это значение. По умолчанию 80.
- `MEMORY_PERCENT`: Пороговое значение использования памяти. TeleMonitor отправит оповещение, если использование превысит это значение. По умолчанию 80.
- `DISK_PERCENT`: Пороговое значение использования диска. TeleMonitor отправит оповещение, если использование превысит это значение. По умолчанию 80.
- `MONITOR_INTERVAL`: Интервал времени (в секундах) для мониторинга. По умолчанию 60.
- `DISK_PATH`: Путь к диску. По умолчанию `/host`.
- `URL_LIST`: Список URL-адресов для мониторинга, разделенных запятыми. По умолчанию пусто.
- `LANGUAGE`: Язык. По умолчанию `en`. Варианты `en` и `cn`.

Вот команды для бота:

- /start - Получить помощь
- /help - Получить помощь
- /status - Получить состояние системы
